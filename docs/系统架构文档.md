# Excel处理服务系统架构文档

## 1. 系统概述
Excel处理服务是一个面向企业运营场景的任务编排与Excel处理平台，支持导入、导出、任务跟踪与外部系统接入。

## 2. 目标用户
- 平台管理员：配置任务定义、客户端、列定义与数据源。
- 外部系统：通过开放API创建任务并获取执行结果。

## 3. 技术栈
- 后端：Spring Boot 3、Spring Security、MyBatis Plus
- 前端：Vue 3 + Element Plus
- 数据库：MySQL / H2
- 存储：MinIO

## 4. 分层架构
- Controller层：REST API接口暴露与参数接收。
- Service层：业务逻辑与流程编排。
- Mapper层：MyBatis Plus数据访问。
- Entity/DTO层：数据库实体与传输对象。

## 5. 核心模块
- 任务管理
- 任务定义管理
- 客户端管理
- 数据源管理
- 列定义管理
- Excel导入导出
- 看板统计

## 6. 数据流
1. 管理界面用户（`username/password`）或外部客户端（`clientId/clientSecret`）先通过登录接口获取凭证。
2. 调用方携带`X-API-Key`访问业务API。
3. 认证过滤器校验“启用客户端”或“有效管理用户”身份。
4. Controller调用Service执行业务。
5. Service通过Mapper访问数据库。
6. 对关键操作写入`operation_log`审计日志。

## 7. 部署架构
- 开发环境：单机运行，H2内存数据库。
- 生产环境：Spring Boot + MySQL + MinIO，敏感配置通过环境变量注入。

## 8. 扩展性设计
- 数据源扩展：支持SQL与HTTP拉取模型。
- 权限扩展：基于`@RequirePermission`注解接入更多细粒度权限。
- 审计扩展：`@OperationLog`可覆盖更多接口实现统一审计。

## 9. 安全设计
- 双登录入口认证（UI用户登录 + SDK客户端登录）
- API Key认证
- RBAC权限检查
- 全局异常处理统一响应
- CI安全扫描（Dependency-Check + CodeQL）
